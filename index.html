Grade 11 Computer Science

My Python journey started a while ago, but I have only begun organizing and documenting my projects when taking Grade 11 Computer Science.

First Print Statements and Basic Operations

In Grade 11 Computer Science, we started with the basics of Python. We reviewed how to print statements and make basic operations. Though this was something I was familar with, it was still fun to create small projects with this knowledge. Programs that were the most fun to create using basic print statements and operations were ones that created art. Who knew that you could create houses, shapes, and even faces using just characters and operations!?


Variables, Inputs, and Outputs

The journey continues with variables, and taking basic inputs and outputs from users. Now I was able to interact with the users and create small, fun games using the knowledge of inputs and operations. Since I enjoyed the art I created in the previous unit, a fun game that I created was asking the user to input the shape and size they wanted, and printed it. I made similar projects to these, experimenting with the different applications I can make, like finance and other calculators.


Conditional Controls

Let the journey go on! We then moved onto if/else statements. Now I was able to add multitasking to the projects I created. I was able to create applications that asked the user questions, and acted depending on their answers. When learning Python on my own, I was familar how to do this, but with the knowledge in class, I was able to learn how to implement if/else statements in smart ways, especially through efficient nesting if/else statement strategies.


Loops

Loops, loops, and more loops! Loops are definitely one of my favourite aspects in Python, since they are extermely useful and quick to use! We started to learn about while and for loops, which can be implemented in many of the existing programs to replace repeating if/else statements. It's almost like magic! I learned how to solve problems with both types of loops, which definitely expanded my knowledge on when to use which loops.


Strings

This was definitely one of the shorter units in Grade 11 Computer Science, since of course we could not spend time going through every single String Method to exist, unless we wanted the class to fall asleep. Instead, in this unit we reviewed the most commonly used string methods, such as format and index. In my own time, I spent time going through different string methods that could be useful for my existing applications.


Lists

It's time for lists! Another short unit in the Grade 11 Computer Science course, but of course, still a very fun one! Who knew that something as simple as lists containing multiple items could be so helpful? I used lists in many of my applications as a method of storing information from the user (before I knew what databases where, of course). Depending on the application, I used lists differently, whether I was splitting it, manipulating the data within, or accessing it.


Functions

The final unit of Grade 11 Computer science, at last! Of course, the unit of functions was the longest unit of all, since now we needed to implement every unit from the course into functions. I knew how to make simple functions, but with the knowledge learned in class, I was able to create more complex functions that were efficient, used less code, and worked for a variety of applications. I even learned how to create functions nested within each other with my knowledge of nested if/else statements!


Final Summative

Time to battle the big beast! The final summative that was given at the end of the Grade 11 Computer Science course was definitely a journey, a very long two-week, staring at my computer screen for hours, journey. We were tasked with creating an alpha to decimal, and vise versa, calculator that followed a very detailed, specific set of rules. Though it took me all that time, I learned so much! I was able to mix together all my skills from the previous units to complete a working application! All the hours sitting on my desk working at this long problem paid off, and definitely granted me a great load of self satisfaction!

Grade 12 Computer Science

Moving on to what I learned in Grade 12 Computer Science! This course taught me about a lot of concepts in Python I was unaware of, and would not have discovered and experimented with without this course. Not only did I extend and strengthen the knowledge and skills I learned in Grade 11 and also on my own, but I also got to learn new skills and create fun projects that allowed me to express my creativity.

Extension on Lists

Though the beginning of the Grade 12 Computer Science course was mainly review from Grade 11, such as review on functions, string methods, loops, etc., I will not be discussing these concepts on this page as it can all be found on the Grade 11 page. However, a topic that we expanded on that really interested me was the versatility of lists. In this course, we covered multidimensional lists, and how this simple concept can be used for so many programs (eg. creating board games, tables, etc.). I got the chance to create fun games, such as a battleship equivalent game.


Dictionaries

After learning about lists, the course covered dictionaries! I was aware of dictionaries beforehand, I did not use them as often, but now I got the chance to extend my knowledge on how to use dictionaries. Creating the dictionaries creates easier ways to access data, retrieve, and delete information. Instead of having to go through multiple lists and rely on indexes, I can take advantage of the speed and efficiency of dictionaries. I was able to create a lot of fun projects, like an item stocking program, and a simple login interface.


Expectation Handling

Another quick topic covered in the course that I felt the need to mention, as it has changed the way I program nowadays, is using exception handling! To keep things sweet and simple, I learned about the easier way to catch errors using try/except, and the multiple errors you can catch to further give users instructions. I use this method of catching mistakes through my programs now, but some examples of how I used try/except could be seen in the examples above.


Recursions

Who knows the concept of calling functions within functions could save so much time! Recursions were definitely one of the topics that took me the most time to understand, but once I did, it was mind blowing! By using diagrams and drawing out how the functions within each other worked, I was able to complete complex operations using as little code as possible. Examples you can see of how I used recursions include computing a factorial sequence, the fibonacci sequence, and other mathematical operations.


Reading/Writing Text Files

This was a unit in the course we covered the most, as reading/writing text files was used when creating apps to hold and retrieve user data. Though there were not a lot of functions when reading and writing text files, since you could store the data in whatever way you would like to, text files could be used for so many different programs! Some fun programs I created including a login interface, grades application, sports score calculator, etc. I even used text files in future units in the course, which can be viewed through programs linked for the units below.


Binary Files

Similar to the structure of text files, we also covered binary files. Though binary files are more secure and used for actual applications to hide information, for simplicity, we used text files more rather than binary files. However, the knowledge of using binary files, and the pickle module used, is something I used when creating my own applications. I created an Employee Record Management system to secure and hide the data of employees, and in the future I plan on creating similar applications or modifying my existing applications using text files to use secure binary files instead.


CSV Files

I found the topic of CSV files to be very similar to text files, and the importing, writing, appending, etc., strategies were very similar. The use of Comma Separated Values made it very easy to fetch data and translate it to use and manipulate, which I found made my programs quicker. That being said, to practice my hand at using CSV files to store data, I replicated my programs using text files but with CSV files. I created another sports score calculator but using CSV files, and also made a temperature application to store daily temperatures.


SQLITE Databases

This unit made my programs so much easier to work with! The ability to instead of having multiple files but now adding them all into one file and linking them together, is so helpful! There is great accessibility with SQLITE databases, and instead of having to work through multiple files of data to make manipulations or additions, now I could just access the tables within the database and make my changes. I made some smaller SQLITE databases, like a student tracker to link marks and students, and an organization tracker to link employees and their information, and there are still some applications I am currenting working on. Though I know how to use SQLITE databases, the knowledge is endless, and there are still features I am researching to further improve my programs.


Object-Oriented Programming (OOP)

There was so much to learn within this unit, and still is a lot to discover! Object-Oriented Program really improved my structure and brushed-up on my programming style. I think after learning about OOP, I have become faster, efficient, and productive when programming. I am able to troubleshoot my mistakes faster, reuse my code through the concept of inheritance, and better structure my data using classes. Within the course, we learned about magic methods to make my life easier when programming, and even how to manipulate them to do what we want. Due to the global access, code organization, and overall independence, I was able to create programs without having to repeat writing, such as area calculators, vector calculator, system of equations calculator, and so many other applications that I would write with regular functions, but now could be written with OOP. I still use the structure of OOP because I enjoy the organization and efficiency, and thus this unit within the course has definitely impacted my skills as a programmer.

Technical Documentation Page

When I was working on a project to create a webpage about writing a good essay, I learned a lot about using HTML for technical documentation. HTML was the main tool I used to structure the webpage and make it easy to navigate. One thing that stood out was making a side menu bar. With HTML, I figured out how to create a menu that stayed on the side of the webpage. This menu had links to different parts of the essay writing guide, like 'Introduction,' 'Body,' 'Conclusion,' and Parts of Essays.'
